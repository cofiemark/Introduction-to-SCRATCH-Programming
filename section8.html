Learning Through Examples

Example projects and step-by-step tutorials are powerful tools for learning Scratch programming. They provide concrete demonstrations of concepts and techniques that students can follow, modify, and extend.

Benefits of Example Projects:

  • Concrete Models: Show practical applications of abstract concepts
  • Scaffolded Learning: Break down complex projects into manageable steps
  • Inspiration: Provide ideas and starting points for original projects
  • Debugging Practice: Help students learn to identify and fix errors
  • Remixing Opportunities: Allow students to build upon existing work

In this section, we'll explore several example projects that demonstrate different aspects of Scratch programming and STEM applications.

Tutorial 1: Simple Animation

This tutorial demonstrates how to create a basic animation using costume changes and movement.

Project Goal:

Create a character that walks across the screen with a realistic walking animation.

Step 1: Set Up the Sprite

  1. Choose a character sprite from the library or draw your own
  2. Make sure the sprite has multiple costumes that show different walking positions
  3. If needed, create additional costumes by duplicating and modifying existing ones

Step 2: Create the Walking Animation

when green flag clicked
go to x: [-200] y: [0]
forever
next costume
change x by [5]
wait [0.2] seconds
if <
[x position] > [240]
> then
go to x: [-200] y: [0]

Step 3: Add Direction Changes

when green flag clicked
go to x: [-200] y: [0]
set rotation style [left-right]
forever
next costume
change x by [5]
wait [0.2] seconds
if <
[x position] > [240]
> then
point in direction [-180]
if <
[x position] < [-240]
> then
point in direction [0]

Extensions:

  • Add a changing backdrop to create a scrolling effect
  • Include sound effects for footsteps
  • Add user controls to change the character's speed or direction

Tutorial 2: Interactive Story

This tutorial demonstrates how to create a simple interactive story with branching paths.

Project Goal:

Create a story where the user makes choices that affect the outcome.

Step 1: Plan Your Story

  1. Create a flowchart of your story with decision points
  2. Prepare backdrops for different scenes
  3. Create or select sprites for characters

Step 2: Set Up the Introduction

when green flag clicked
switch backdrop to [scene1]
say [Welcome to the adventure!] for [3] seconds
say [You find yourself at a crossroads...] for [3] seconds
ask [Do you go left or right? (type "left" or "right")] and wait

Step 3: Create Branching Paths

if <
[answer] = [left]
> then
switch backdrop to [forest]
say [You enter a dense forest...] for [3] seconds
ask [Do you follow the path or explore? (type "path" or "explore")] and wait
if <
[answer] = [path]
> then
switch backdrop to [cabin]
say [You find a cozy cabin!] for [3] seconds
else
switch backdrop to [cave]
say [You discover a mysterious cave...] for [3] seconds
else
switch backdrop to [river]
say [You arrive at a flowing river...] for [3] seconds
ask [Do you swim across or look for a bridge? (type "swim" or "bridge")] and wait
if <
[answer] = [swim]
> then
switch backdrop to [waterfall]
say [You're swept away to a beautiful waterfall!] for [3] seconds
else
switch backdrop to [village]
say [You find a bridge that leads to a village!] for [3] seconds

Extensions:

  • Add more decision points and outcomes
  • Include variables to track the user's choices
  • Add character sprites that appear in different scenes
  • Include sound effects and music for different scenes

Tutorial 3: Simple Game - Catch the Apple

This tutorial demonstrates how to create a simple game with scoring and timing.

Project Goal:

Create a game where the player controls a basket to catch falling apples.

Step 1: Set Up the Sprites

  1. Create or select a basket sprite for the player to control
  2. Create or select an apple sprite that will fall from the top
  3. Choose a backdrop for your game

Step 2: Program the Basket Movement

when green flag clicked
go to x: [0] y: [-140]
forever
set x to [mouse x]
if <
[x position] < [-230]
> then
set x to [-230]
if <
[x position] > [230]
> then
set x to [230]

Step 3: Program the Falling Apple

when green flag clicked
set [score] to [0]
set [time] to [30]
forever
go to x: [pick random [-230] to [230]] y: [180]
repeat until <
[y position] < [-170]
or
touching [Basket]?
>
change y by [-5]
if <
touching [Basket]?
> then
change [score] by [1]
play sound [pop]

Step 4: Add Timer and Game Over

when green flag clicked
set [time] to [30]
repeat until <
[time] = [0]
>

Tutorial 4: STEM Project - Angle Simulator

This tutorial demonstrates how to create a mathematical tool for exploring angles.

Project Goal:

Create an interactive angle demonstrator that allows users to adjust angles and see their measurements.

Step 1: Set Up the Project

  1. Create a sprite for the angle vertex (a small circle)
  2. Create a sprite for the angle arm (a line)
  3. Create a plain backdrop with a coordinate grid

Step 2: Program the Fixed Arm

when green flag clicked
go to x: [0] y: [0]
switch costume to [horizontal-line]
set rotation style [all around]
point in direction [0]

Step 3: Program the Movable Arm

when green flag clicked
set [angle] to [0]
go to x: [0] y: [0]
switch costume to [line]
set rotation style [all around]
point in direction [0]
forever
point in direction [angle]
say [join [Angle: ] [join [angle] [°]]]

Step 4: Add User Controls

when [right arrow] key pressed
change [angle] by [5]
if <
[angle] > [360]
> then
set [angle] to [
[angle] - [360]
]
when [left arrow] key pressed
change [angle] by [-5]
if <
[angle] < [0]
> then
set [angle] to [
[angle] + [360]
]

Step 5: Add Real-World Examples

when [space] key pressed
if <
[angle] = [90]
> then
say [90° is a right angle, like the corner of a room or a piece of paper.] for [3] seconds
else
if <
[angle] = [180]
> then

else
if <
[angle] = [360]
or
[angle] = [0]
> then
say [360° (or 0°) is a full circle, like a wheel or a clock.] for [3] seconds

Extensions:

  • Add a protractor image that aligns with the angle
  • Include calculations for complementary and supplementary angles
  • Add trigonometric functions (sine, cosine, tangent) of the angle
  • Create a game where students must create specific angles
