Assessment in Scratch Programming

Assessing student learning in Scratch programming requires approaches that go beyond traditional testing. Project-based assessment allows educators to evaluate not just what students know, but what they can create and how they think.

Key Assessment Principles:

  • Process-Oriented: Evaluate the development process, not just the final product
  • Authentic: Assess through real-world projects and applications
  • Formative: Provide ongoing feedback throughout the learning process
  • Reflective: Encourage students to reflect on their own learning
  • Comprehensive: Consider technical skills, creativity, and computational thinking

In this section, we'll explore strategies for assessing student learning in Scratch programming and tracking their progression through increasingly complex concepts.

Project-Based Assessment

Project-based assessment evaluates student learning through the creation of Scratch projects that demonstrate their understanding and skills.

Assessment Strategies:

  • Project Rubrics: Define clear criteria for evaluating projects
  • Portfolio Assessment: Evaluate a collection of projects over time
  • Peer Review: Have students evaluate each other's work
  • Project Presentations: Ask students to explain their projects and process
  • Code Reviews: Examine the scripts students create for efficiency and clarity

Sample Project Rubric Categories:

  • Functionality: Does the project work as intended?
  • Complexity: Does the project use appropriate programming concepts?
  • Creativity: Does the project show original thinking?
  • Design: Is the project visually appealing and user-friendly?
  • Documentation: Is the project well-explained and commented?
  • Process: Did the student follow an effective development process?

Example Rubric for a Game Project

Criteria Beginning (1) Developing (2) Proficient (3) Exemplary (4)
Functionality Game doesn't work or has major bugs Game works but has minor bugs Game works as intended with no significant bugs Game works flawlessly with additional features
Programming Concepts Uses only basic sequences Uses loops and simple conditionals Uses variables and complex conditionals Uses advanced features like lists, clones, or custom blocks
User Interface Confusing or difficult to use Basic but functional interface Clear, intuitive interface Polished, engaging interface with helpful feedback

Assessing Computational Thinking

Computational thinking is a set of problem-solving skills that are fundamental to computer science but applicable across disciplines. Scratch is designed to develop these skills.

Key Computational Thinking Skills:

  • Decomposition: Breaking down complex problems into smaller parts
  • Pattern Recognition: Identifying similarities or patterns
  • Abstraction: Focusing on important information while ignoring irrelevant details
  • Algorithmic Thinking: Developing step-by-step solutions
  • Debugging: Finding and fixing errors

Assessment Strategies for Computational Thinking:

  • Problem-Solving Tasks: Present challenges that require computational thinking
  • Code Analysis: Have students explain how and why code works
  • Debugging Exercises: Provide broken code for students to fix
  • Project Planning: Evaluate how students plan and structure their projects
  • Reflection Questions: Ask students to reflect on their problem-solving process

Sample Reflection Questions

  1. How did you break down this problem into smaller parts?
  2. What patterns did you notice that helped you solve the problem?
  3. How did you decide what details were important and what could be ignored?
  4. Explain your step-by-step solution to this problem.
  5. What errors did you encounter, and how did you fix them?

Progression Pathways

Progression pathways outline how students can advance from basic to advanced programming concepts in a structured way.

Levels of Progression:

  1. Level 1: Basic Motion and Events
    • Using motion blocks to move sprites
    • Responding to simple events (green flag, key press)
    • Creating simple sequences of instructions
  2. Level 2: Loops and Simple Animation
    • Using repeat loops for repetitive actions
    • Creating animations with costume changes
    • Using wait blocks for timing
  3. Level 3: Conditionals and User Input
    • Using if/else blocks for decision-making
    • Responding to user input (keyboard, mouse)
    • Using sensing blocks to detect conditions
  4. Level 4: Variables and Simple Games
    • Creating and using variables to store data
    • Implementing scoring systems
    • Creating simple game mechanics
  5. Level 5: Complex Logic and Coordination
    • Using nested loops and conditionals
    • Broadcasting messages between sprites
    • Implementing more complex game mechanics
  6. Level 6: Advanced Features
    • Using lists to store collections of data
    • Creating clones for multiple instances
    • Defining custom blocks for reusable code
  7. Level 7: Complex Projects and Extensions
    • Creating multi-level games or simulations
    • Using extensions for additional functionality
    • Implementing complex algorithms

Each level builds upon the previous ones, introducing new concepts while reinforcing existing skills.

Debugging and Problem-Solving

Debugging—finding and fixing errors in code—is an essential skill in programming. Assessing students' debugging abilities provides insight into their problem-solving skills and understanding of programming concepts.

Debugging Assessment Strategies:

  • Bug-Finding Exercises: Provide code with intentional errors for students to identify
  • Fix-It Tasks: Give students broken projects to repair
  • Error Explanation: Ask students to explain why errors occur and how to fix them
  • Debugging Process Documentation: Have students document their debugging process
  • Peer Debugging: Have students find and fix errors in each other's projects

Sample Debugging Exercise

The following script is supposed to make a sprite move in a square pattern, but it doesn't work correctly. Identify and fix the errors:

when green flag clicked
repeat [4]
move [100] steps
turn left [90] degrees

Possible errors to identify:

  • The turn direction might need to be "right" instead of "left" depending on the desired pattern
  • The script might need a "pen down" block to make the square visible
  • The script might need a starting position reset

Self-Assessment and Reflection

Encouraging students to assess their own work and reflect on their learning process helps develop metacognitive skills and independence.

Self-Assessment Strategies:

  • Project Checklists: Provide criteria for students to check their own work
  • Learning Journals: Have students document their learning process
  • Before-and-After Comparisons: Ask students to compare their current and previous work
  • Goal Setting: Encourage students to set and evaluate their own learning goals
  • Peer Feedback: Facilitate constructive feedback between students

Sample Self-Assessment Questions

  1. What was the most challenging part of this project, and how did you overcome it?
  2. What programming concepts did you use in this project?
  3. What would you do differently if you were to create this project again?
  4. How does this project demonstrate your growth as a programmer?
  5. What new skills or concepts would you like to learn next?
