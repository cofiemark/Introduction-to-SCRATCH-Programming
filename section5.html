Understanding Sprites

Sprites are the characters and objects that perform actions in Scratch projects. They are a fundamental element of any Scratch program.

Key Characteristics of Sprites:

  • Independent Objects: Each sprite has its own scripts, costumes, and sounds
  • Properties: Sprites have properties like position, direction, size, and visibility
  • Interaction: Sprites can interact with other sprites, the stage, and user input
  • Multiple Costumes: Each sprite can have multiple appearances (costumes)

The sprite properties panel showing position, direction, and other attributes

Creating and Managing Sprites

Scratch offers several ways to create and manage sprites in your projects.

Adding New Sprites:

  • Choose from Library: Select from hundreds of pre-made sprites
  • Paint: Draw your own sprite using the built-in paint editor
  • Upload: Import an image file from your computer
  • Surprise: Get a random sprite from the library
  • Camera: Take a photo with your webcam to create a sprite

Managing Sprites:

  • Duplicate: Create a copy of an existing sprite, including its scripts
  • Delete: Remove a sprite from the project
  • Export: Save a sprite to your computer for use in other projects
  • Hide/Show: Control sprite visibility during project execution
  • Resize: Change the size of a sprite

The Sprite Pane at the bottom of the Scratch interface shows thumbnails of all sprites in your project. Click on a sprite to select it and edit its properties, scripts, costumes, or sounds.

Understanding Costumes

Costumes are the different appearances that a sprite can have. By switching between costumes, you can create animations and visual changes in your sprites.

Key Concepts:

  • Multiple Appearances: Each sprite can have multiple costumes
  • Animation: Switching costumes creates animation effects
  • Costume Library: Access pre-made costumes for your sprites
  • Costume Editor: Modify costumes using the built-in paint tools

The costume editor in Scratch

Working with the Paint Editor

The Paint Editor is a built-in tool for creating and editing sprite costumes and stage backdrops.

Paint Editor Features:

  • Vector and Bitmap Modes: Choose between scalable vector graphics or pixel-based bitmap graphics
  • Drawing Tools: Brush, line, rectangle, circle, and text tools
  • Color Palette: Select colors for your drawings
  • Fill Tool: Fill enclosed areas with color
  • Selection Tools: Select, move, resize, and rotate parts of your costume
  • Layers: Work with multiple layers in vector mode

Vector vs. Bitmap Mode:

  • Vector Mode: Creates scalable graphics that maintain quality when resized; better for simple shapes and drawings
  • Bitmap Mode: Creates pixel-based graphics; better for detailed images and photos

You can switch between vector and bitmap modes using the buttons at the bottom of the Paint Editor, but be aware that converting from one mode to the other may affect the quality of your image.

Creating Animations

Animation in Scratch is primarily achieved by switching between costumes. By changing costumes rapidly, you can create the illusion of movement.

Types of Animations:

  • Frame-by-Frame: Create multiple costumes, each representing a frame of animation
  • Movement: Combine costume changes with position changes for more complex animations
  • Interactive: Change costumes in response to user input or other events

Example: Simple Walking Animation

when green flag clicked
forever
next costume
move [10] steps
wait [0.2] seconds

This script creates a walking animation by cycling through costumes while moving the sprite. The wait block controls the speed of the animation.

Example: Animation Based on Direction

when [right arrow] key pressed
switch costume to [facing right]
change x by [10]
when [left arrow] key pressed
switch costume to [facing left]
change x by [-10]

This example changes the sprite's costume based on which direction it's moving, creating a more realistic movement effect.

Sprite Positioning and Rotation

Controlling a sprite's position and rotation is essential for creating dynamic projects.

Position:

  • Coordinates: Sprites are positioned using x and y coordinates
  • Origin: The stage's origin (0,0) is at the center
  • X-axis: Ranges from -240 (left) to 240 (right)
  • Y-axis: Ranges from -180 (bottom) to 180 (top)

Rotation:

  • Direction: Sprites have a direction property (0-359 degrees)
  • Rotation Style: Controls how sprites appear when their direction changes:
    • All around: Sprite rotates to match its direction
    • Left-right: Sprite flips horizontally but doesn't rotate
    • Don't rotate: Sprite maintains its orientation regardless of direction

Example: Moving in a Circle

when green flag clicked
go to x: [0] y: [0]
forever
move [10] steps
turn right [5] degrees

This script makes a sprite move in a circular pattern by repeatedly moving forward and turning slightly.

Working with Backdrops

Backdrops are the images that appear on the stage behind the sprites. They set the scene for your project and can change during execution.

Adding Backdrops:

  • Choose from Library: Select from pre-made backdrops
  • Paint: Create your own backdrop using the paint editor
  • Upload: Import an image file from your computer
  • Camera: Take a photo with your webcam

Switching Backdrops:

switch backdrop to [backdrop2]
next backdrop
when backdrop switches to [backdrop2]

Changing backdrops can be used to create different scenes in a story, levels in a game, or visual feedback for different states in your project.

Example: Scene Changes in a Story

when green flag clicked
switch backdrop to [scene1]
say [Welcome to my story!] for [2] seconds
wait [3] seconds
switch backdrop to [scene2]
say [Let's continue our adventure...] for [2] seconds

This script creates a simple story with scene changes by switching backdrops and displaying dialogue.
