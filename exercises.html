Practice Makes Perfect

The best way to learn programming is through hands-on practice. This page provides a collection of exercises and activities designed to reinforce the concepts covered in each section of the course. These exercises range from simple tasks for beginners to more complex challenges for advanced learners.

How to Use These Exercises:

  • Progressive Learning: Start with exercises that match your current skill level
  • Regular Practice: Aim to complete at least one exercise after each section
  • Experimentation: Don't be afraid to modify exercises and try your own ideas
  • Reflection: After completing an exercise, think about what you learned and how you might apply it
  • Collaboration: Many exercises can be done in pairs or small groups

Each exercise includes a description, learning objectives, step-by-step instructions, and extension challenges for those who want to go further.

Beginner Exercises

These exercises are designed for those who are new to Scratch and programming.

Exercise 1: Hello, Scratch!

Objective: Create a simple animation with a sprite that introduces itself.

Skills Practiced: Basic motion, looks blocks, simple sequencing

Instructions:

  1. Choose a sprite from the library or draw your own
  2. Make the sprite move across the screen
  3. Have the sprite say "Hello, I'm [sprite name]!"
  4. Make the sprite change color or size
  5. End with a simple animation (like spinning or dancing)

Extension:

Add a second sprite that interacts with the first one, such as having a conversation.

Exercise 2: Interactive Drawing

Objective: Create a simple drawing program where the sprite follows the mouse and draws.

Skills Practiced: Pen blocks, mouse interaction, conditionals

Instructions:

  1. Choose a small sprite to be your "pen"
  2. Make the sprite follow the mouse pointer
  3. When the mouse button is pressed, put the pen down
  4. When the mouse button is released, lift the pen up
  5. Add a way to clear the drawing (e.g., press the space key)

Extension:

Add buttons to change the pen color and size.

Exercise 3: Counting Game

Objective: Create a simple game that asks the user to count objects.

Skills Practiced: Variables, user input, conditionals

Instructions:

  1. Create a variable called "correct answer"
  2. Display a random number (1-10) of the same sprite on the screen
  3. Ask the user "How many [objects] do you see?"
  4. Check if their answer matches the correct number
  5. Provide feedback on whether they were right or wrong

Extension:

Add a scoring system and multiple rounds.

Intermediate Exercises

These exercises are for those who have mastered the basics and are ready for more complex projects.

Exercise 4: Maze Game

Objective: Create a game where a sprite navigates through a maze.

Skills Practiced: Keyboard control, collision detection, game design

Instructions:

  1. Draw a maze as the backdrop or using sprites for walls
  2. Create a player sprite that can be controlled with arrow keys
  3. Program the sprite to stop when it touches a wall
  4. Add a starting point and a goal
  5. Create a timer to track how long it takes to complete the maze

Extension:

Add collectible items, obstacles, or multiple levels.

Exercise 5: Quiz Show

Objective: Create an interactive quiz on a STEM topic.

Skills Practiced: Variables, conditionals, user input, broadcasting

Instructions:

  1. Create a list of at least 5 questions and answers on a STEM topic
  2. Design a quiz show interface with a host sprite
  3. Use the "ask and wait" block to get user answers
  4. Keep score based on correct answers
  5. Provide feedback after each question

Extension:

Add multiple-choice options, different difficulty levels, or a timer for each question.

Exercise 6: Virtual Pet

Objective: Create a virtual pet that responds to user interactions.

Skills Practiced: Variables, conditionals, costume changes, user interaction

Instructions:

  1. Design a pet sprite with multiple costumes showing different emotions
  2. Create variables for hunger, happiness, and energy
  3. Add buttons or keyboard controls to feed, play with, and let the pet rest
  4. Program the pet to change appearance based on its needs
  5. Make the variables change over time (e.g., hunger increases)

Extension:

Add more complex behaviors, growth stages, or mini-games to play with your pet.

Advanced Exercises

These exercises challenge students to apply multiple concepts and create more sophisticated projects.

Exercise 7: Physics Simulator

Objective: Create a simple physics simulation demonstrating concepts like gravity and collision.

Skills Practiced: Variables, mathematical operations, cloning, complex motion

Instructions:

  1. Create a ball sprite that responds to gravity
  2. Program the ball to bounce realistically off surfaces
  3. Add variables for gravity strength, bounce elasticity, and friction
  4. Create controls to adjust these variables
  5. Add the ability to create multiple balls using cloning

Extension:

Add different types of surfaces (bouncy, sticky), obstacles, or implement air resistance.

Exercise 8: Ecosystem Simulation

Objective: Create a simulation of a simple ecosystem with predators and prey.

Skills Practiced: Cloning, variables, lists, complex interactions

Instructions:

  1. Create sprites for plants, herbivores, and carnivores
  2. Use cloning to create multiple instances of each
  3. Program basic behaviors: plants grow, herbivores eat plants, carnivores eat herbivores
  4. Add variables for energy, reproduction, and population tracking
  5. Create graphs to display population changes over time

Extension:

Add environmental factors like weather, seasons, or natural disasters that affect the ecosystem.

Exercise 9: Interactive Data Visualization

Objective: Create an interactive visualization of a data set related to a STEM topic.

Skills Practiced: Lists, mathematical operations, user interface design

Instructions:

  1. Choose a data set related to a STEM topic (e.g., planetary data, climate data)
  2. Store the data in lists
  3. Create a visual representation of the data (bar chart, scatter plot, etc.)
  4. Add interactive elements that allow users to explore the data
  5. Include explanations of what the data shows

Extension:

Add the ability to input or modify data, or create multiple visualization types.

STEM-Specific Challenges

These challenges focus specifically on applying Scratch to different STEM disciplines.

Mathematics Challenge: Geometric Explorer

Objective: Create an interactive tool for exploring geometric concepts.

Skills Practiced: Mathematical operations, custom blocks, user interface

Instructions:

  1. Create custom blocks for drawing different geometric shapes
  2. Add controls to adjust parameters (sides, angles, size)
  3. Display mathematical properties of the shapes (perimeter, area)
  4. Allow users to combine shapes to create patterns
  5. Include educational information about each shape

Extension:

Add 3D shapes, transformations (rotation, reflection), or coordinate geometry.

Science Challenge: Solar System Model

Objective: Create an accurate model of the solar system with planets orbiting the sun.

Skills Practiced: Mathematical operations, scaling, research skills

Instructions:

  1. Research the relative sizes and orbital distances of planets
  2. Create sprites for the sun and each planet, scaled appropriately
  3. Program each planet to orbit at the correct relative speed
  4. Add information about each planet that appears when clicked
  5. Include controls to speed up or slow down the simulation

Extension:

Add moons, asteroid belts, or the ability to view from different perspectives.

Engineering Challenge: Bridge Builder

Objective: Create a simulation where users design and test bridges.

Skills Practiced: Physics concepts, user interface design, variables

Instructions:

  1. Create a scene with two platforms separated by a gap
  2. Allow users to "build" bridge segments between the platforms
  3. Implement simple physics to test if the bridge can support weight
  4. Add variables for materials, strength, and cost
  5. Create a challenge mode with specific constraints

Extension:

Add different bridge types (suspension, arch), environmental factors (wind, earthquakes), or more realistic physics.

Technology Challenge: Encryption Tool

Objective: Create a tool that demonstrates basic encryption concepts.

Skills Practiced: String operations, algorithms, user interface

Instructions:

  1. Create an interface where users can input a message
  2. Implement a simple encryption algorithm (like Caesar cipher)
  3. Display both the original and encrypted messages
  4. Add the ability to decrypt messages
  5. Include an explanation of how the encryption works

Extension:

Implement multiple encryption methods, add a key system, or create a challenge where users have to decrypt messages.

Collaborative Projects

These projects are designed for small groups to work on together, encouraging collaboration and communication.

Project 1: Interactive Story Adventure

Objective: Create a multi-scene interactive story with branching paths.

Group Size: 2-4 students

Instructions:

  1. As a group, plan a story with multiple possible paths and endings
  2. Divide responsibilities: story writing, character design, programming, backgrounds
  3. Create a system for tracking the user's choices and progress
  4. Implement scene transitions and character interactions
  5. Test the story with other groups and refine based on feedback

Collaboration Tips:

Use Scratch's backpack feature to share sprites and scripts between group members. Hold regular meetings to ensure all parts work together.

Project 2: STEM Fair Exhibit

Objective: Create an interactive exhibit that teaches a STEM concept.

Group Size: 3-5 students

Instructions:

  1. Choose a STEM concept that can be demonstrated interactively
  2. Research the concept thoroughly to ensure accuracy
  3. Design an engaging interface with clear instructions
  4. Create interactive elements that help users understand the concept
  5. Include assessment questions to check user understanding

Collaboration Tips:

Assign specific roles based on strengths: content expert, programmer, designer, user experience tester. Create a project plan with milestones.

Project 3: Game Design Challenge

Objective: Design and create an educational game related to a STEM topic.

Group Size: 2-4 students

Instructions:

  1. Identify a STEM concept that would benefit from gamification
  2. Design game mechanics that reinforce learning the concept
  3. Create engaging gameplay with increasing levels of difficulty
  4. Implement a scoring system and feedback mechanisms
  5. Test with peers and iterate based on feedback

Collaboration Tips:

Break the game into components that can be developed independently, then integrate. Use version control by saving numbered versions of your project.

Assessment Challenges

These challenges can be used as assessment activities to evaluate student understanding and skills.

Challenge 1: Debug the Code

Objective: Find and fix errors in provided Scratch projects.

Assessment Focus: Debugging skills, understanding of programming concepts

Instructions:

  1. Provide students with Scratch projects containing intentional errors
  2. Ask them to identify and fix all the bugs
  3. Have them document what each bug was and how they fixed it
  4. Include bugs related to different programming concepts

Evaluation Criteria:

Number of bugs successfully fixed, accuracy of explanations, efficiency of solutions.

Challenge 2: Build from Specifications

Objective: Create a project that meets specific requirements.

Assessment Focus: Following instructions, implementing required features

Instructions:

  1. Provide a detailed specification for a project (e.g., a game with specific features)
  2. Include both required and optional features
  3. Set a time limit for completion
  4. Evaluate based on how well the project meets the specifications

Evaluation Criteria:

Completeness, accuracy, efficiency of code, creativity within constraints.

Challenge 3: Explain Your Code

Objective: Create a project and provide a detailed explanation of how it works.

Assessment Focus: Understanding of concepts, communication skills

Instructions:

  1. Students create a project of their choice using specified programming concepts
  2. They must add comments to their code explaining how it works
  3. They prepare a presentation or written explanation of their project
  4. The explanation should connect to programming and STEM concepts

Evaluation Criteria:

Clarity of explanation, accuracy of terminology, depth of understanding, quality of project.
