Overview of Block Categories

Scratch organizes its programming blocks into categories, each with a distinct color and purpose. This organization helps users quickly find the blocks they need and understand their function.

The block palette showing different categories of blocks

The main block categories in Scratch 3.0 are:

  • Motion (Blue): Control sprite movement and position
  • Looks (Purple): Change appearance and visibility
  • Sound (Pink): Play and control audio
  • Events (Yellow): Respond to triggers and start scripts
  • Control (Orange): Manage script flow with loops and conditionals
  • Sensing (Light Blue): Detect conditions and get input
  • Operators (Green): Perform mathematical and logical operations
  • Variables (Orange/Red): Store and retrieve data
  • My Blocks (Pink/Red): Create custom blocks

Additional categories may appear when extensions are added to a project.

Motion Blocks

Motion blocks control how sprites move around the stage. They are colored blue and are among the most commonly used blocks in Scratch.

The Motion blocks category

Key Motion Blocks:

move [10] steps

Moves the sprite forward in its current direction by the specified number of steps (pixels).

turn right [15] degrees
turn left [15] degrees

Rotates the sprite clockwise or counterclockwise by the specified angle.

go to x: [0] y: [0]

Moves the sprite to the specified coordinates on the stage.

glide [1] secs to x: [0] y: [0]

Moves the sprite smoothly to the specified coordinates over the given time period.

point in direction [90]

Sets the sprite's direction (0 = up, 90 = right, 180 = down, 270 = left).

point towards [mouse-pointer]

Makes the sprite face toward the mouse pointer or another sprite.

if on edge, bounce

Makes the sprite bounce when it hits the edge of the stage.

Motion blocks are essential for creating animations, games, and interactive stories where characters need to move around the stage.

Looks Blocks

Looks blocks control the appearance of sprites. They are colored purple and allow you to change costumes, create speech bubbles, apply visual effects, and control size and visibility.

The Looks blocks category

Key Looks Blocks:

say [Hello!] for [2] seconds
say [Hello!]

Creates a speech bubble with the specified text, either temporarily or until changed.

think [Hmm...] for [2] seconds
think [Hmm...]

Creates a thought bubble with the specified text.

switch costume to [costume2]
next costume

Changes the sprite's appearance by switching costumes.

switch backdrop to [backdrop1]
next backdrop

Changes the stage's background.

change size by [10]
set size to [100]%

Adjusts the sprite's size.

change [color] effect by [25]
set [color] effect to [0]

Applies visual effects like color, fisheye, whirl, etc.

clear graphic effects

Removes all visual effects from the sprite.

show
hide

Controls the sprite's visibility.

Looks blocks are crucial for creating visual feedback, character dialogue, and visual transformations in projects.

Sound Blocks

Sound blocks control audio playback and effects. They are colored pink and allow you to play sounds, adjust volume, and create music.

The Sound blocks category

Key Sound Blocks:

play sound [meow] until done

Plays the selected sound and waits for it to finish before continuing the script.

start sound [meow]

Starts playing the sound but immediately continues the script without waiting.

stop all sounds

Stops all currently playing sounds.

change [pitch] effect by [10]
set [pitch] effect to [100]

Adjusts sound effects like pitch, pan, or echo.

clear sound effects

Removes all sound effects.

change volume by [-10]
set volume to [100]%

Controls the volume of sounds.

volume

Reports the current volume level.

Sound blocks enhance projects by adding audio feedback, music, sound effects, and voice recordings.

Events Blocks

Events blocks trigger scripts based on specific occurrences. They are colored yellow and typically appear at the top of scripts as starting points.

The Events blocks category

Key Events Blocks:

when green flag clicked

Starts the script when the green flag button is clicked.

when [space] key pressed

Starts the script when the specified key is pressed.

when this sprite clicked

Starts the script when the sprite is clicked.

when backdrop switches to [backdrop1]

Starts the script when the backdrop changes to the specified one.

when [loudness] > [10]

Starts the script when a sensor value exceeds the threshold.

when I receive [message1]

Starts the script when the specified broadcast message is received.

broadcast [message1]
broadcast [message1] and wait

Sends a message to all sprites, triggering "when I receive" scripts.

Events blocks are fundamental to making projects interactive, allowing them to respond to user input and other triggers.

Control Blocks

Control blocks manage the flow of scripts. They are colored orange and include loops, conditionals, and timing controls.

The Control blocks category

Key Control Blocks:

wait [1] seconds

Pauses the script for the specified number of seconds.

repeat [10]

Runs the enclosed blocks a specific number of times.

forever

Continuously runs the enclosed blocks until the script is stopped.

if <> then

Runs the enclosed blocks only if the condition is true.

if <> then ... else

Runs one set of blocks if the condition is true, another set if false.

wait until <>

Pauses the script until the condition becomes true.

repeat until <>

Repeats the enclosed blocks until the condition becomes true.

stop [all]

Stops scripts (all scripts, this script, or other scripts in sprite).

when I start as a clone

Starts the script when a clone of the sprite is created.

create clone of [myself]

Creates a duplicate of the specified sprite.

delete this clone

Removes the current clone.

Control blocks are essential for creating complex behaviors, managing timing, and implementing game logic.

Sensing Blocks

Sensing blocks detect conditions and gather input. They are colored light blue and provide information about the environment and user interactions.

The Sensing blocks category

Key Sensing Blocks:

touching [edge]?
touching [Sprite2]?

Detects if the sprite is touching the edge, another sprite, or a color.

touching color [#FF0000]?
color [#FF0000] is touching [#0000FF]?

Detects color collisions.

distance to [mouse-pointer]

Reports the distance to the mouse pointer or another sprite.

ask [What's your name?] and wait

Prompts the user for input and waits for a response.

answer

Reports the user's most recent answer.

key [space] pressed?
mouse down?

Detects keyboard and mouse input.

mouse x
mouse y

Reports the current mouse coordinates.

loudness

Reports the volume level from the microphone.

timer
reset timer

Measures elapsed time.

current [year]

Reports date and time information.

Sensing blocks are crucial for creating interactive projects that respond to user actions and environmental conditions.

Operators Blocks

Operators blocks perform mathematical and logical operations. They are colored green and help with calculations, text manipulation, and comparisons.

The Operators blocks category

Key Operators Blocks:

[3] + [4]
[8] - [2]
[3] * [4]
[8] / [2]

Perform basic arithmetic operations.

pick random [1] to [10]

Generates a random number within the specified range.

<[10] < [20]>
<[10] = [10]>
<[10] > [5]>

Compare values and return true or false.

<<[10] < [20]> and <[5] > [2]>>
<<[10] < [20]> or <[5] < [2]>>
<not <[10] < [20]>>

Perform logical operations (AND, OR, NOT).

join [hello ] [world]

Combines text strings.

letter [1] of [world]

Reports a specific character from a text string.

length of [hello world]

Reports the number of characters in a text string.

[9] mod [4]

Reports the remainder when dividing the first number by the second.

round [3.14]

Rounds a number to the nearest integer.

[abs] of [-5]
[sqrt] of [16]

Perform mathematical functions.

Operators blocks are essential for calculations, decision-making, and text manipulation in projects.

Variables Blocks

Variables blocks allow you to create, store, and manipulate data. They are colored orange/red and include both variables and lists.

The Variables blocks category

Key Variables Blocks:

set [my variable] to [0]

Assigns a value to a variable.

change [my variable] by [1]

Increases or decreases a variable's value.

show variable [my variable]
hide variable [my variable]

Controls the visibility of a variable monitor on the stage.

add [thing] to [my list]

Adds an item to the end of a list.

delete [1] of [my list]

Removes an item from a list.

insert [thing] at [1] of [my list]

Adds an item at a specific position in a list.

replace item [1] of [my list] with [thing]

Changes an item in a list.

item [1] of [my list]

Reports the value of a specific item in a list.

length of [my list]

Reports the number of items in a list.

[my list] contains [thing]?

Checks if a specific item is in a list.

Variables blocks are crucial for creating games with scores, simulations with changing values, and any project that needs to store and track data.

My Blocks (Custom Blocks)

My Blocks allow you to create your own reusable blocks. They are colored pink/red and help organize code and create abstractions.

The My Blocks category

Creating Custom Blocks:

  1. Click "Make a Block" in the My Blocks category
  2. Name your block and optionally add parameters
  3. Define what the block does by attaching other blocks to the "define" block
define [draw square [size]]

Creates a new custom block, potentially with parameters.

Custom blocks are powerful for:

  • Breaking down complex scripts into manageable parts
  • Reusing code in multiple places
  • Creating higher-level abstractions
  • Making scripts more readable and maintainable
