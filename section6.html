Understanding Interactivity

Interactivity is what makes Scratch projects engaging and dynamic. Interactive projects respond to user input and create a two-way experience between the user and the program.

Types of Interactivity in Scratch:

  • User Input: Responding to keyboard, mouse, or microphone input
  • Conditional Responses: Changing behavior based on user actions
  • Feedback: Providing visual or auditory feedback to user actions
  • Dialogue: Creating conversations or prompts for user input
  • Game Mechanics: Implementing rules, scoring, and challenges

In this section, we'll explore how to create projects that respond to various forms of user input and provide engaging interactive experiences.

Keyboard Interaction

Keyboard interaction allows users to control sprites and trigger events using keyboard keys.

Key Blocks for Keyboard Interaction:

when [space] key pressed

Starts a script when the specified key is pressed.

key [space] pressed?

Reports whether a specific key is currently being pressed (true/false).

Example: Simple Character Movement

when [right arrow] key pressed
change x by [10]
when [left arrow] key pressed
change x by [-10]
when [up arrow] key pressed
change y by [10]
when [down arrow] key pressed
change y by [-10]

This example creates a character that can be moved in four directions using the arrow keys.

Example: Continuous Movement with Key Sensing

when green flag clicked
forever
if <
key [right arrow] pressed?
> then
change x by [5]
if <
key [left arrow] pressed?
> then
change x by [-5]

This approach creates smoother, continuous movement by checking key states in a loop rather than using separate event handlers.

Mouse Interaction

Mouse interaction allows users to control projects using mouse movements and clicks.

Key Blocks for Mouse Interaction:

when this sprite clicked

Starts a script when the sprite is clicked.

mouse down?

Reports whether the mouse button is currently pressed (true/false).

mouse x
mouse y

Report the current x and y coordinates of the mouse pointer.

touching [mouse-pointer]?

Reports whether the sprite is touching the mouse pointer (true/false).

point towards [mouse-pointer]

Makes the sprite point in the direction of the mouse pointer.

go to [mouse-pointer]

Moves the sprite to the position of the mouse pointer.

Example: Drag and Drop

when green flag clicked
forever
if <
mouse down?
and
touching [mouse-pointer]?
> then
go to [mouse-pointer]

This script allows the user to drag a sprite around the stage by clicking and holding on it.

Example: Interactive Button

when this sprite clicked
switch costume to [button-pressed]
wait [0.2] seconds
switch costume to [button-normal]
broadcast [button-clicked]

This example creates a button that visually responds when clicked and broadcasts a message that can trigger actions in other sprites.

User Input with Ask and Answer

The "ask and answer" blocks allow your projects to prompt users for text input and use their responses.

ask [What's your name?] and wait

Displays a text prompt and waits for the user to type a response.

answer

Reports the user's most recent answer.

Example: Personalized Greeting

when green flag clicked
ask [What's your name?] and wait
say [Hello, ]
join [Hello, ] [answer]
[!] for [2] seconds

This script asks for the user's name and then displays a personalized greeting.

Example: Simple Quiz

when green flag clicked
ask [What is 5 + 3?] and wait
if <
[answer] = [8]
> then
say [Correct!] for [2] seconds
else
say [Sorry, the answer is 8.] for [2] seconds

This example creates a simple math quiz that checks the user's answer and provides feedback.

Creating Interactive Stories

Interactive stories allow users to influence the narrative through their choices and actions.

Key Elements of Interactive Stories:

  • Branching Narratives: Different paths based on user choices
  • Character Dialogue: Text displayed using say/think blocks
  • Scene Changes: Switching backdrops to represent different locations
  • User Choices: Allowing users to make decisions that affect the story

Example: Simple Branching Story

when green flag clicked
switch backdrop to [forest]
say [You're lost in a forest. Do you go left or right?] for [4] seconds
ask [Type "left" or "right"] and wait
if <
[answer] = [left]
> then
switch backdrop to [cave]
say [You found a mysterious cave...] for [3] seconds
else
switch backdrop to [river]
say [You arrived at a flowing river...] for [3] seconds

This example creates a simple interactive story where the user's choice determines which path the story takes.

Game Mechanics

Game mechanics are the rules and systems that create engaging gameplay experiences.

Common Game Mechanics in Scratch:

  • Scoring: Keeping track of points or achievements
  • Lives/Health: Managing player status and game over conditions
  • Timers: Creating time pressure or measuring performance
  • Collision Detection: Detecting when sprites touch each other or specific colors
  • Movement Patterns: Creating predictable or random movement for game elements
  • Levels: Increasing difficulty or changing environments as the game progresses

Example: Simple Collecting Game

when green flag clicked
set [score] to [0]
forever
go to x: [pick random [-240] to [240]] y: [pick random [-180] to [180]]
wait [1] seconds
when green flag clicked
forever
if <
touching [Player]?
> then
change [score] by [1]
go to x: [pick random [-240] to [240]] y: [pick random [-180] to [180]]
play sound [collect]

This example creates a simple game where a collectible item (this sprite) appears in random locations, and the player earns points by touching it.

Example: Timer and Game Over

when green flag clicked
set [time] to [30]
repeat until <
[time] = [0]
>
wait [1] seconds
change [time] by [-1]
say [Game Over!] for [2] seconds
stop [all]

This script creates a countdown timer and ends the game when time runs out.

Creating Effective User Interfaces

A good user interface (UI) makes your project intuitive and enjoyable to use.

UI Design Tips for Scratch Projects:

  • Clear Instructions: Tell users how to interact with your project
  • Visual Feedback: Provide visual cues when users take actions
  • Consistent Controls: Use similar interaction patterns throughout your project
  • Readable Text: Ensure text is large enough and contrasts with the background
  • Intuitive Layout: Organize elements in a logical way

Example: Start Screen with Instructions

when green flag clicked
switch backdrop to [title-screen]
show
wait until <
this sprite clicked
>
switch backdrop to [game-screen]
hide
broadcast [start game]

This script creates a start screen that waits for the user to click before beginning the game.
