Computational Thinking with Scratch

Scratch is designed to develop computational thinking skills, which are valuable across all STEM disciplines. These skills include:

  • Decomposition: Breaking down complex problems into smaller, manageable parts
  • Pattern Recognition: Identifying similarities or patterns among different problems
  • Abstraction: Focusing on important information while ignoring irrelevant details
  • Algorithmic Thinking: Developing step-by-step solutions to problems

In this section, we'll explore how Scratch implements fundamental programming concepts that support computational thinking.

Algorithms and Sequencing

An algorithm is a step-by-step procedure for solving a problem or accomplishing a task. In Scratch, algorithms are created by connecting blocks in a specific order.

Key Concepts:

  • Sequence: The order in which instructions are executed
  • Precision: Instructions must be clear and unambiguous
  • Determinism: The same input should produce the same output

Example: Drawing a Square

when green flag clicked
pen down
move [100] steps
turn right [90] degrees
move [100] steps
turn right [90] degrees
move [100] steps
turn right [90] degrees
move [100] steps
turn right [90] degrees
pen up

This algorithm demonstrates sequencing - each instruction is executed in order from top to bottom. The precision of the instructions (exact number of steps and degrees) ensures the square is drawn correctly.

Loops

Loops allow a sequence of instructions to be repeated multiple times. They are essential for reducing repetition in code and making it more efficient.

Types of Loops in Scratch:

  • Repeat: Executes blocks a specific number of times
  • Forever: Continuously executes blocks until the script is stopped
  • Repeat Until: Repeats blocks until a condition becomes true

Example: Drawing a Square with a Loop

when green flag clicked
pen down
repeat [4]
move [100] steps
turn right [90] degrees
pen up

This improved algorithm uses a loop to repeat the "move and turn" sequence four times, making the code more concise and easier to understand.

Example: Continuous Animation with Forever Loop

when green flag clicked
forever
move [10] steps
if on edge, bounce
next costume
wait [0.1] seconds

This script creates continuous movement and animation using a forever loop, which runs indefinitely until the project is stopped.

Conditionals

Conditionals allow scripts to make decisions based on whether certain conditions are true or false. They enable programs to respond differently in different situations.

Types of Conditionals in Scratch:

  • If: Executes blocks only if a condition is true
  • If-Else: Executes one set of blocks if a condition is true, another set if false
  • Wait Until: Pauses execution until a condition becomes true

Example: Simple If Statement

when green flag clicked
forever
if <
touching [mouse-pointer]?
> then
say [Hello!] for [2] seconds

This script checks if the sprite is touching the mouse pointer. If true, it says "Hello!" for 2 seconds.

Example: If-Else Statement

when green flag clicked
forever
if <
key [space] pressed?
> then
change y by [10]
else
change y by [-5]

This script moves the sprite up when the space key is pressed and down when it's not, creating a simple jumping mechanism with gravity.

Variables

Variables store and retrieve data in a program. They are essential for keeping track of information that changes during program execution.

Using Variables in Scratch:

  • Creating Variables: Click "Make a Variable" in the Variables category
  • Scope: Variables can be for all sprites (global) or just for one sprite (local)
  • Monitoring: Variables can be displayed on the stage to show their current values

Example: Simple Score Counter

when green flag clicked
set [score] to [0]
forever
if <
touching [Apple]?
> then
change [score] by [1]
wait [0.5] seconds

This script initializes a score variable to 0 and increases it by 1 whenever the sprite touches another sprite named "Apple".

Example: Timer

when green flag clicked
set [time left] to [30]
repeat until <
[time left] = [0]
>
wait [1] seconds
change [time left] by [-1]
say [Time's up!] for [2] seconds

This script creates a countdown timer that starts at 30 seconds and counts down to zero, then displays "Time's up!"

Events

Events trigger scripts in response to specific occurrences. They are fundamental to creating interactive programs that respond to user input and other triggers.

Common Events in Scratch:

  • User Input: Key presses, mouse clicks, etc.
  • Project Start: When the green flag is clicked
  • Messages: When a broadcast is received
  • State Changes: When a backdrop changes, when a value exceeds a threshold

Example: Multiple Event Handlers

when green flag clicked
say [Use arrow keys to move me!] for [2] seconds
when [up arrow] key pressed
change y by [10]
when [down arrow] key pressed
change y by [-10]
when [right arrow] key pressed
change x by [10]
when [left arrow] key pressed
change x by [-10]

This example uses multiple event blocks to create a character that responds to arrow key presses by moving in different directions.

Parallelism

Parallelism allows multiple scripts to run simultaneously. This is a powerful concept in Scratch that enables complex behaviors and interactions.

Implementing Parallelism in Scratch:

  • Multiple Scripts: Different scripts can run at the same time
  • Multiple Sprites: Each sprite can have its own independent scripts
  • Broadcasts: Messages can trigger multiple scripts across different sprites

Example: Parallel Scripts

when green flag clicked
forever
move [10] steps
if on edge, bounce
when green flag clicked
forever
next costume
wait [0.2] seconds

This example has two scripts that run in parallel: one handles movement and bouncing, while the other handles animation by changing costumes.

Example: Broadcast and Receive

when green flag clicked
wait [2] seconds
broadcast [start game]
when I receive [start game]
set [score] to [0]
forever
move [5] steps

This example uses broadcasting to coordinate between scripts. When the "start game" message is broadcast, it triggers another script to initialize the game and start movement.

Debugging in Scratch

Debugging is the process of finding and fixing errors in programs. Scratch provides several features that make debugging easier:

  • Highlighting: Active blocks are highlighted as they run
  • Single-stepping: Click on a script to run it step by step
  • Variable monitors: Display variable values on the stage
  • Say blocks: Use say blocks to display values during execution

Common Debugging Strategies:

  1. Break down complex scripts into smaller parts
  2. Test each part individually
  3. Use say blocks to display intermediate values
  4. Check conditions in if statements
  5. Verify that loops are terminating properly
