Beyond the Basics

Once you've mastered the fundamental concepts of Scratch programming, you can explore advanced features that enable more sophisticated projects. These features allow for more complex interactions, better organization, and enhanced functionality.

In This Section:

  • Cloning: Creating multiple instances of sprites
  • Lists: Storing and manipulating collections of data
  • Custom Blocks: Creating your own reusable code blocks
  • Broadcasting: Coordinating actions between sprites
  • Extensions: Adding new capabilities to Scratch
  • Cloud Variables: Sharing data between users

These advanced features can significantly enhance your Scratch projects, especially for STEM education applications that require more complex functionality.

Cloning

Cloning allows you to create multiple instances of a sprite during program execution. This is useful for creating multiple similar objects without having to program each one individually.

Key Concepts:

  • Clone Creation: Making copies of sprites during runtime
  • Clone Behavior: Programming how clones behave
  • Clone Management: Creating and deleting clones as needed
create clone of [myself]

Creates a copy of the sprite that runs its own scripts.

when I start as a clone

Starts a script when a clone is created.

delete this clone

Removes the clone from the project.

Example: Fireworks Display

when green flag clicked
hide
forever
wait [pick random [1] to [3]] seconds
go to x: [pick random [-200] to [200]] y: [-150]
create clone of [myself]
when I start as a clone
show
set [speed] to [pick random [5] to [10]]
repeat [20]
change y by [speed]
change [speed] by [-0.5]
repeat [10]
create clone of [myself]
point in direction [pick random [0] to [360]]
change [color] effect by [25]
repeat [20]
move [10] steps
change [ghost] effect by [5]
delete this clone

This example creates a fireworks display by launching "rockets" that explode into multiple particles, each represented by a clone.

Applications in STEM Education:

  • Physics: Simulating particle systems, molecular motion
  • Biology: Modeling cell division, population growth
  • Mathematics: Demonstrating fractals, probability

Lists

Lists allow you to store and manipulate collections of data. They are similar to arrays in other programming languages and enable more complex data handling.

Key Concepts:

  • List Creation: Making lists to store multiple values
  • List Operations: Adding, removing, and changing list items
  • List Access: Retrieving values from specific positions
  • List Properties: Checking length and contents
add [thing] to [my list]

Adds an item to the end of a list.

delete [1] of [my list]

Removes an item from a specific position in the list.

insert [thing] at [1] of [my list]

Adds an item at a specific position in the list.

replace item [1] of [my list] with [thing]

Changes the value of an item at a specific position.

item [1] of [my list]

Reports the value of an item at a specific position.

length of [my list]

Reports the number of items in the list.

[my list] contains [thing]?

Checks if a specific item is in the list.

Example: To-Do List Application

when green flag clicked
delete [all] of [tasks]
forever
ask [Enter a task (or type "view" to see your list):] and wait
if <
[answer] = [view]
> then
if <
[length of [tasks]] = [0]
> then
say [Your to-do list is empty.] for [2] seconds
else
set [index] to [1]
repeat [length of [tasks]]
say [join [index] [join [. ] [item [index] of [tasks]]]] for [2] seconds
change [index] by [1]
else
add [answer] to [tasks]
say [Task added!] for [1] seconds

This example creates a simple to-do list application where users can add tasks and view their list.

Applications in STEM Education:

  • Data Science: Collecting and analyzing data
  • Mathematics: Working with sequences, sets
  • Computer Science: Implementing algorithms that use data structures

Custom Blocks

Custom blocks (also called "My Blocks") allow you to create your own reusable code blocks. They help organize code, reduce repetition, and create higher-level abstractions.

Key Concepts:

  • Block Definition: Creating new blocks with specific functionality
  • Parameters: Adding inputs to customize block behavior
  • Abstraction: Hiding implementation details behind a simple interface
  • Reusability: Using the same block in multiple places
define [draw square [size]]

Creates a new custom block, potentially with parameters.

Example: Drawing Shapes

define [draw polygon [sides] [size]]
pen down
set [angle] to [
[360] / [sides]
]
repeat [sides]
move [size] steps
turn right [angle] degrees
pen up
when green flag clicked
draw polygon [4] [100]
change x by [150]
draw polygon [6] [50]
change y by [-150]
draw polygon [3] [100]

This example creates a custom block for drawing regular polygons with a specified number of sides and size, then uses it to draw different shapes.

Applications in STEM Education:

  • Mathematics: Creating procedures for geometric constructions
  • Computer Science: Teaching procedural abstraction and modularity
  • Engineering: Building reusable components for simulations

Broadcasting

Broadcasting allows sprites to communicate with each other by sending and receiving messages. It's useful for coordinating actions between different parts of your project.

Key Concepts:

  • Message Sending: Broadcasting a message to all sprites
  • Message Receiving: Responding to broadcast messages
  • Synchronization: Coordinating actions between sprites
  • Event-Driven Programming: Responding to events rather than following a fixed sequence
broadcast [message1]

Sends a message to all sprites and continues immediately.

broadcast [message1] and wait

Sends a message and waits for all triggered scripts to complete.

when I receive [message1]

Starts a script when the specified message is received.

Example: Multi-Character Story

when green flag clicked
switch backdrop to [scene1]
say [Once upon a time...] for [3] seconds
broadcast [character1 enter]
wait [5] seconds
broadcast [character2 enter]
wait [5] seconds
broadcast [scene change]
when I receive [character1 enter]
go to x: [-150] y: [-100]
show
say [Hello there!] for [3] seconds
when I receive [scene change]
switch backdrop to [scene2]
broadcast [continue story]

This example uses broadcasting to coordinate a story with multiple characters and scene changes.

Applications in STEM Education:

  • Biology: Modeling cell signaling, ecosystem interactions
  • Physics: Simulating action-reaction pairs, wave propagation
  • Computer Science: Teaching event-driven programming, message passing

Extensions

Extensions add new capabilities to Scratch, allowing you to work with external hardware, access additional features, or use specialized blocks for specific purposes.

Available Extensions:

  • Music: Play instruments and drum sounds
  • Pen: Draw with sprites using advanced pen features
  • Video Sensing: Detect motion in video from a camera
  • Text to Speech: Convert text to spoken audio
  • Translate: Translate text between languages
  • Micro:bit: Connect to BBC micro:bit hardware
  • LEGO MINDSTORMS: Control LEGO robots
  • LEGO Education WeDo 2.0: Control LEGO WeDo devices
  • makey makey: Interact with Makey Makey input devices

Example: Using the Pen Extension

when green flag clicked
clear
pen up
go to x: [0] y: [0]
set pen color to [#FF0000]
set pen size to [5]
pen down
set [sides] to [6]
set [length] to [100]
repeat [sides]
move [length] steps
turn right [
[360] / [sides]
] degrees

This example uses the Pen extension to draw a regular polygon with a specified number of sides.

Applications in STEM Education:

  • Robotics: Programming physical devices
  • Art and Design: Creating complex drawings and patterns
  • Linguistics: Exploring language translation and speech synthesis
  • Physical Computing: Connecting digital and physical worlds

Cloud Variables

Cloud variables allow data to be shared between different users running the same Scratch project. They enable collaborative projects and persistent data storage.

Key Concepts:

  • Cloud Storage: Storing data on Scratch servers
  • Shared Access: Multiple users accessing the same variables
  • Real-time Updates: Changes by one user visible to others
  • Persistence: Data remains available between sessions

To create a cloud variable, prefix the variable name with the cloud symbol (☁) when creating it. Cloud variables are only available to users who are logged in to Scratch.

Example: Collaborative Counter

when green flag clicked
say [This project has been run ]
☁ counter
[ times!] for [2] seconds
change [☁ counter] by [1]

This simple example increments a cloud variable each time the project is run, allowing users to see how many times the project has been started by all users.

Applications in STEM Education:

  • Citizen Science: Collecting data from multiple users
  • Collaborative Research: Sharing findings between students
  • Social Studies: Conducting surveys and analyzing results
  • Game Design: Creating multiplayer experiences
